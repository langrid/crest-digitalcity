新しいマシンで FreeWalk をコンパイルする時

FreeWalk のソースコードの中で QvLib という VRML パーサを利用していますが、 デフォルトではこの QvLib のインクルードファイルを読み込むことができません。 (「インクルード ファイルがオープンできません」というエラーが出ます)
このエラーを出さないようにするためには次の設定を行なって下さい。

  1. 「ツール」メニューバーの「オプション」を開く
  2. 「ディレクトリ」の項目を選択する
  3. 「表示するディレクトリ」を「インクルードファイル」にする
  4. Qvlib のインクルードファイルが入っているディレクトリを追加する(下図)
デバッグの方法

普通の Windows アプリケーションの場合はブレークポイントなどを 利用することができますが、 FreeWalk は ActiveX コントロールを利用しているので、この方法は使えません。 また、デバッグには「ActiveX コントロール テストコンテナ」を利用します。 Developer Studio 内で「実行」すると多分これが起動されるはずです。 その後、「編集」-「新しいコントロールの挿入」で ActiveX コントロールをテストすることができます。

デバッグ用の文字列や数字などを出したい

先ほどの「ActiveX コントロール テストコンテナ」内でも OutputDebugString を使えばデバッグ用の文字列の出力が可能です。

また、FreeWalk では DPF というデバッグ用のマクロが用意されています。

 例 : DPF("%d,%d\n", x, y);

利用するためには Comm/debug.h をインクルード必要があります また、Developer Studio 内の「ビルド」メニューの「アクティブな構成の設定」で、 「Fwvr - Win32 Debug」を選択しなければいけません。 (そうしないとデバッグ文字列が出力されない)。
