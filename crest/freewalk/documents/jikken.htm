 1.会話

 

 今回の実験は何れもAgentとAvatarが共存し、相互に会話が可能であることが特徴である。

 従って何れもAgentとAvatarが共存した状態で実験を行う。

 

 Agentの会話データ蓄積

 Agent　は[現在のAvatarの集団による迷路探索歩行実験および地下鉄駅内の探索歩行実験]によって会話データを蓄積する。

 

 AgentとAvatarの会話

 Agent  は　Avatar　に話しかける；　Avatar　は　Agent　に話し言葉で答える。

 Avatar　は　Agent　に話しかける；　Agent　は　Avatar　に話し言葉で答える。

 

                            誘導エージェント                                          アバタ−

セリフ                  録音された少数のセリフを連呼                     バリエーションのある発話内容

会話理解              基本的に無し（「出口」などの単語に反応）  内容を理解

 2.建築空間における会話を含む群集流動シミュレーション

 

 FREEWALKと群集流動シミュレーションの両方の特徴を生かすためには、AgentとAvatarの会話により、Avatarあるいは群集が誘導されて避難するような行動のシミュレーションを行う。そのためには、以下のような例が考えられる。

 

 1)　地下鉄今出川駅の避難計画

 地下2階のプラットフォームに到着した満員の車両から(人数の想定が必要：別資料参照)、プラットフォーム、コンコースを通り、地上へ避難する。このときにAgentがAvatarや群集のAgentに話しかけ、誘導する。また、AvatarもAgentや他のAvatarに話しかけて道を尋ねる。

 

 地下鉄電車→地下2階プラットフォーム→階段・エスカレータ→改札口→地下1階コン コース→地上への階段・エレベータ

 

脱出経路上に正しい脱出方向が一見して分からない混乱地点（分岐点や広場）を数カ所用意する．車両から出た群集（避難エージェント＋実験者のアバター数人）は「出口はあっちだ！」などと口々に叫びながら，各混乱地点においては大部分（９割）が間違った方向へ避難する．各混乱地点では，群集と同じ格好をした一般人，もしくは駅員（誘導エージェント，もしくはアバタ−）が正しい方向に誘導しようと試みる．この誘導がどれくらい成功するかによって，インタフェースの違い（一般人vs駅員）及びエージェントの種類（ソフトウェアvs人間）が社会的信用に与える影響をテストする．

 

条件１：一般人    （誘導エージェント）

条件２：駅員       （誘導エージェント）

条件３：一般人    （アバタ−）

条件４：駅員       （アバタ−）

 

信用度に関する仮説１：条件１　＜　条件２　＜　条件３　＜　条件４

 2)迷路での探索誘導歩行実験（迷路脱出と集団成因の協調、競合）

　AvatarとAgentの会話により、迷路の中を誘導されて目標に到達する群集の歩行シミュレーション。　AvatarはAgentまたは他のAvatarに道を尋ねる。AgentはAvatarに話しかけ、案内する。

　

　迷路は小人数なので実験がしやすい。また会話も多様な状況で行いやすい。

 


地下鉄での実験に先立って，単純な迷路を用いた誘導実験を行う．ここでは，群集を用いず，一人のアバタ−に迷路を解かせる．スタートからゴールまでの間に分岐点を５つほど用意し，各分岐点に誘導員（誘導エージェントもしくはアバタ−）を立たせる．被験者には誘導員が時々嘘をつくと教授した上で，二人目以降の誘導員は間違ったアドバイスを与えるようにする．何人の誘導員を信用するかをテストすることで，誘導エージェントとアバタ−の社会的信用の違いを見る．

 3)　京都駅での群集流動

 コンコースが避難階で正面に出口があるため、避難行動シミュレーションには適していない。また、空間の見通しも良いため、案内や道を尋ねる会話も発生しにくい。 切符売り場で切符を買ったり、改札口を通り、コンコースを横切る群集の歩行シ ミュレーションは可能と考えられるが、これだけでは、従来の群集流動シミュレーシ ョンと違いが無い。

 　従って、FREEWALKと群集流動シミュレーションを組み合わせた意義を示せるような 実験とはなりにくい。　

　京都駅の特徴は全体的な立面にある。特に高い場所にある。歩行者が他の歩行者を見る視野（シミュレーションの視野角度は小さい）には全体像や特徴的な場所が入りにくい。

　研究的性質が希薄。

 また、VRMLデータ作成に膨大な時間を要する。
