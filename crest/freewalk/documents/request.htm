要求仕様

1.        サーバ

l         クライアントが表示する画像データ（3D空間データ、人体データ）を蓄積できること

l         クライアントからの音声データを受信できること

l         音声合成ができること

l         合成した音声データをクライアントに送信できること

l         クライアントからの映像データを受信できること

l         映像データをクライアントに送信できること

l         クライアント・群集エージェント・誘導エージェントの衝突判定ができること（クライアント・群集エージェント・誘導エージェントの移動ベクトルデータを一元的に持つ機能を有すること）

l         クライアント・群集エージェント・誘導エージェントの描画情報（座標データ）を各クライアントに送信できること

l         新規にクライアントが参加してきた際、そのクライアントの情報を各クライアントに送信できること（メンバ管理情報をもつこと）

 

2.        クライアント

       画像

l         ３D空間をInternet Explorer上に表示できること

l         ３D空間はVRML2.0データであること

l         VRML2.0データをOpenGLデータに変換できること

l         群集エージェント（100名程度）とクライアント（4名程度）、誘導エージェント（5名程度）が存在した際、表示速度が遅くないこと

l         サーバから描画用データ(空間データ・人体データ)をダウンロードできること

l         カメラから映像を入力し、Internet Explorer上のクライアント画面に表示できること

l         カメラから入力した映像をサーバに送信できること

l         サーバから他のクライアントのカメラ映像を受信できること

l         ジェスチャーデータを受信できること

l         ジェスチャーデータをアニメ変換できること

l         アニメ変換されたジェスチャーデータを表示できること

 

       音声

l         マイクから音声入力ができること

l         マイクから入力した音声を形態素解析できること

l         サーバに音声を送信できること

l         サーバから送られてきた音声データを受信できること

l         音声通信に遅延がないこと（別スレッドにて対応）

 

       操作

l         キーボード入力で移動できること

l         サーバに移動ベクトルを送信できること

 

       人体モデルアニメーション

l         歩行アルゴリズムにしたがって体の各関節角度を決定できること

l         歩行アニメーションの処理結果に基づいて人体を描写できること

 

3.        避難エージェント

l         群集歩行プログラムを元に各人の座標と向きを計算できること

l         サーバに各人の移動ベクトルを送信できること

l         WAVEファイルが再生できること

l         サーバに再生したWAVEファイルの音声データを送信できること

 

4.        誘導エージェント

l         形態素解析された音声データを受信できること

l         サーバに位置データを送信できること

l         WAVEファイルが再生できること

l         サーバに再生したWAVEファイルの音声データを送信できること

l         ジェスチャーデータを送信できること
