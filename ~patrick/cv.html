3 Research Interests (detail)

Currently, my main interest lies in multi-platform network application design; that is, building network applications that are seamlessly or easily portable. I am specifically interested in the design and implementation of applications based on reusable modular components implemented to best suit any available relevant platform.
As a broader range of network devices becomes available, modular and portable design becomes critical in building applications that can take advantage of the best aspects of the available platforms while still maintaining a clear functionality throughout the various implementations. Data availability is at the core of any network application, but can represent a particular challenge when platforms range from powerful servers and workstations to mobile phones and refrigerators.
There is currently no single toolkit for building meaningful multi-platform network applications. Java, web technologies, and database software together provide a good starting place but leave many problems, including those with multimedia, interactive and mobile content, unsolved.

Kawaramachi-Shijo E-commerce Project
E-commerce improvement project for merchants in the Kawaramachi-Shijo shopping district of Kyoto. The project currently involves creating a virtual community for over 1000 merchants in the area and representing that community as one store in internet space. Elements of the community should include access to: individual merchants; their products; relevent area related information such as local temples, shrines, and historic place; realtime interaction with representatives of the community; and easy horizontal navigation through all of the content in the virtual community.

6 Past Projects

Modular Distributed Network Applications (MTV Networks Online)
Design and implementation of internet based network applications with an emphasis on scalability and portability. The primary focus of this project was creating an architecture upon which a diverse variety of internet services could be built with hardware and software components that could easily be shared among the different services. Services included www, email, rich media delivery, and client/server systems for data access by remote offices and partners.
Technologies/Methods used: Java, Perl, TCP/IP, web server, database, systems optimization, flow control, functionality evaluation and consolidation

Systems Optimization (Sonicnet)
Explored methods for best use of limited hardware resource set in delivery of dynamic, database driven web applications. Objective of reliable performance of independent services within a single system environment was achieved through systematic separation of system resources.
Technologies/Methods used: Solaris, Sun hardware, load/stress testing, benchmarking
Digital Manhattan Interactive Map (Parsons School of Design)
Consolidation of information relating directly to the island of Manhattan. Prototype was built consisting of physical and virtual interfaces to multimedia presentations. During the course of the project, a representative set of local information was selected and collected. From there, appropriate media was chosen for the individual aspects of the collected data and presentations were fabricated. Finally, all the prepared work was unified in a physical interface that represented data access points geographically. Presented at SIGGRAPH.
Technologies/Methods used: Unix, Macintosh, Windows, Alias, SoftImage, Shockwave, mTroplis, Video, Audio
